COMO PROBAR EL PROGRAMA
-----------------------
Para probar este sistema necesitas un servidor web que soporte PHP y MySQL. Tienes dos opciones principales:

OPCIÓN A: PROBAR EN HOSTINGER (RECOMENDADA)
   Esta es la opción real donde vivirá la página.
   1. Sube los archivos a la carpeta `public_html` de tu Hostinger.
   2. Crea la base de datos MySQL en el panel de Hostinger.
   3. Entra a phpMyAdmin e importa el archivo `database.sql` (que ahora crea tablas 'usuarios', 'turnos', 'pagos', etc).
   4. Configura el archivo `.env` con los datos de esa base de datos.
   5. Navega a tu dominio (ej: tusitio.com) y prueba registrarte.

OPCIÓN B: PROBAR LOCALMENTE (EN TU COMPUTADORA)
   Si quieres probarlo en tu PC antes de subirlo, necesitas instalar un programa que simule un servidor.
   1. Descarga e instala **Laragon** (recomendado para Windows) o **XAMPP**.
   2. Inicia Laragon/XAMPP (Apache y MySQL).
   3. Copia la carpeta del proyecto a `C:\laragon\www\aplicacionWebPsicologaNaza` (o htdocs en XAMPP).
   4. Instala las dependencias de Laravel. Abre la terminal en la carpeta y ejecuta:
      `composer install`
      `npm install`
      `npm run build` (o copy el css manualmente como hicimos).
   5. Crea una base de datos local llamada `psicologa_db` usando phpMyAdmin local (http://localhost/phpmyadmin).
   6. Importa el archivo `database.sql`.
   7. Configura el `.env` local:
      - YA HE CREADO ESTE ARCHIVO POR VOS con tus datos (DB_PORT=3307, etc).
      - Solo falta un paso: abre la terminal en la carpeta y ejecuta: `php artisan key:generate`
      - Esto completará la configuración de seguridad.
   8. Abre el navegador en `http://aplicacionWebPsicologaNaza.test` (si usas Laragon) o `http://localhost/aplicacionWebPsicologaNaza/public`.

--------------------------------------------------------------

PASOS TÉCNICOS DE INSTALACIÓN (SERVER DEPLOYMENT)

1. PREPARACIÓN
   - Asegúrate de que la versión de PHP sea 8.1+.

2. INTEGRACIÓN
   - Reemplaza/Agrega las carpetas `app/`, `resources/` y `routes/` que te he enviado.

3. BASE DE DATOS ESPAÑOL
   - Importa `database.sql`. Verifica que se creen las tablas: `usuarios`, `turnos`, `pagos`, `disponibilidades`.

4. CONFIGURACIÓN (.env)
   - DB_CONNECTION=mysql
   - MAIL_MAILER=smtp (Hostinger settings)

5. USUARIOS
   - Regístrate como paciente.
   - Para ser ADMIN: Edita la tabla `usuarios` y cambia el campo `rol` de 'paciente' a 'admin'.

6. ALMACENAMIENTO
   - Ejecuta `php artisan storage:link` para que se vean las imágenes de los pagos.
